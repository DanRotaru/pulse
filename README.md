# pulse

A real-time monitoring dashboard for your machine and your AI coding assistants.

It serves a single-page Vue 3 app from an Express server and streams live data
over Server-Sent Events (SSE):

- **System metrics** — CPU load %, logical core count, RAM used/total, and the
  top 5 processes by CPU and by memory. Refreshed every 2 seconds.
- **AI usage** — Claude Code and Codex rate-limit utilization (5-hour and 7-day
  windows) with reset times, fetched live from the providers' OAuth usage
  endpoints using your locally logged-in credentials. Refreshed every 5 minutes.

The dashboard is password-protected and binds to all network interfaces by
default, so you can open it from your phone or another PC on the same network.

## Architecture

```
Browser (Vue 3 SPA)  ──EventSource('/sse')──▶  Express server (index.js)
        │                                              │
        │  GET /api/stats, /api/usage                  ├─ stats.js   → systeminformation (CPU/RAM/procs)
        └──────────────────────────────────────────────┤  usage.js   → Claude + Codex OAuth usage APIs
                                                        └─ express-sse → broadcasts `stats` + `usage` events
```

| File              | Responsibility                                                                       |
| ----------------- | ------------------------------------------------------------------------------------ |
| `index.js`        | Express server: auth, static hosting, SSE broadcast loop, JSON API endpoints         |
| `stats.js`        | Collects CPU / RAM / top-process metrics via `systeminformation` (with caching)      |
| `usage.js`        | Fetches Claude & Codex rate-limit usage (live OAuth APIs, with local-log fallback)   |
| `src/`            | Vue 3 frontend (`App.vue`, `Dashboard.vue`, `main.js`, `style.css`)                  |
| `vite.config.js`  | Vite build (outputs to `public/`) + dev proxy for `/sse` and `/api` to `:3000`       |
| `public/`         | Built frontend bundle served by Express (generated by `npm run build`, git-ignored)  |
| `static/`         | Static source assets (favicon, etc.) copied into the build                           |

## How usage data is read

The server reads credentials that the CLIs already store locally — nothing is
sent anywhere except to the respective provider's usage API:

- **Claude** — OAuth token from `~/.claude/.credentials.json`, queried against
  `https://api.anthropic.com/api/oauth/usage`.
- **Codex** — token from `~/.codex/auth.json`, queried against the ChatGPT
  backend usage API (auto-refreshing the access token when expired). If the live
  API is unreachable, it falls back to scanning local `~/.codex/sessions` logs.

If you aren't logged in to one of the tools, its card simply shows that usage is
unavailable — the rest of the dashboard still works.

## Requirements

- Node.js `^22.18.0 || >=24.12.0` (the start scripts use `--env-file-if-exists`,
  available in Node 22.9+)

## Setup

```sh
npm install
```

Create your local environment file from the example and edit the values:

```sh
cp .env.example .env
```

### Environment variables

All are optional — sensible defaults are baked into `index.js` if no `.env` is
present. See `.env.example`.

| Variable             | Default     | Description                                                                    |
| -------------------- | ----------- | ------------------------------------------------------------------------------ |
| `PORT`               | `3000`      | Port the Express server listens on                                             |
| `DASHBOARD_PASSWORD` | `12345`     | Password required to view the dashboard and call the API/SSE endpoints         |
| `HOST`               | `0.0.0.0`   | Interface to bind to. `0.0.0.0` = reachable on your LAN; `127.0.0.1` = local   |
| `PROC_INTERVAL_MS`   | `5000`      | How often the heavy RAM + per-process snapshot is refreshed (ms)               |

`.env` is git-ignored; `.env.example` is committed as a template.

## Running

### Production (build once, serve with Express)

```sh
npm run build    # bundles the Vue app into public/
npm run start    # starts the Express server (loads .env if present)
```

On Windows you can also double-click / run `a.bat`, which just calls
`npm run start`.

The server prints the local URL and any reachable LAN URLs on startup:

```
Server listening on http://localhost:3000
  on your network: http://192.168.x.x:3000
```

Use `npm run start:watch` to auto-restart the server on file changes (requires
`nodemon`).

### Development (hot-reload frontend)

Run the API server and the Vite dev server side by side:

```sh
npm run start    # terminal 1 — Express API + SSE on :3000
npm run dev      # terminal 2 — Vite dev server with HMR
```

Vite proxies `/sse` and `/api` to the Express server on `:3000` (see
`vite.config.js`), so the frontend talks to the live backend while you develop.

## Authentication

Every API and SSE endpoint requires the password. The frontend prompts for it,
then the server sets an `httpOnly` cookie (`pulseAuth`, 1-year expiry) so you
stay signed in.

| Method   | Endpoint           | Purpose                                                  |
| -------- | ------------------ | -------------------------------------------------------- |
| `GET`    | `/api/auth`        | Verify the password and set the auth cookie              |
| `DELETE` | `/api/auth`        | Log out (clears the cookie)                              |
| `GET`    | `/sse`             | SSE stream emitting `stats` and `usage` events           |
| `GET`    | `/api/stats`       | One-shot system metrics snapshot                         |
| `GET`    | `/api/usage`       | One-shot usage for both agents                           |
| `GET`    | `/api/usage/:agent`| Refresh a single agent (`claude` or `codex`) and rebroadcast |

The password can be supplied either via the cookie or an `x-dashboard-password`
request header.
